# Deadlock Randomizer

## 📌 О проекте
**Deadlock Randomizer** — это веб-приложение для генерации случайных сборок (предметов и способностей) для игры Deadlock от Valve.
Суть проекта заключается в том, чтобы разнообразить игровой опыт пользователей, предлагая им собирать персонажей по сгенерированным "рулеткой" билдам. В этой версии проекта представлен классический (базовый) рандомайзер без интеграции машинного обучения.

## ⚙️ Стек технологий
* **Backend:** Python, Django
* **Frontend:** HTML5, CSS3 (Vanilla), JavaScript
* **Синхронизация:** Django Channels (WebSockets) для синхронизации лобби в реальном времени.
* **Стилизация:** Кастомный дизайн, имитирующий визуальный стиль игры (шрифты, цвета, эффекты наведения).

## 🎮 Как это работает
1. Пользователь заходит на сайт и видит карточку своего профиля.
2. Он может пригласить друзей по специальной ссылке (или через код лобби), и они появятся в интерфейсе рядом благодаря WebSocket-синхронизации.
3. Игрок выбирает своего героя из выпадающего списка.
4. Нажимает кнопку **"ROLL ALL"** (Сгенерировать всё) или **"ROLL ITEM"** (Сгенерировать один предмет).
5. Алгоритм берет базу из всех предметов игры (загруженную из открытого API) и случайным образом заполняет инвентарь игрока (Weapon, Vitality, Spirit).
6. Если игрок хочет убрать предмет, он может навести на него курсор и нажать `✕`.
7. Все действия по добавлению, удалению и сбросу предметов мгновенно отображаются у всех остальных игроков в лобби.

## ⚠️ Текущие проблемы и ограничения
* **"Слепой" рандом:** В текущей реализации (без ML) алгоритм просто достает предметы из общей корзины. Это приводит к тому, что герою, специализирующемуся на физическом уроне (например, Haze), могут выпасть предметы только на магию (Spirit), что сделает билд неиграбельным и испортит матч.
* **Прокачка скиллов:** Базовый генератор не учитывает реальные синергии прокачки способностей.

*(Примечание: Работа над умным алгоритмом с контролируемой температурой генерации ведется в отдельной ML-ветке).*

## 🚀 Установка и запуск
1. Склонируйте репозиторий: `git clone https://github.com/FerMaxim/deadlock-random-item.git`
2. Создайте и активируйте виртуальное окружение: `python -m venv venv`
3. Установите зависимости: `pip install -r requirements.txt`
4. Выполните миграции: `python manage.py migrate`
5. Запустите сервер: `python manage.py runserver`
6. Откройте в браузере `http://127.0.0.1:8000/`
