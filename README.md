# Deadlock Randomizer (ML Integration Branch)

## 📌 О проекте
**Deadlock Randomizer** — это веб-приложение для генерации сборок (предметы + прокачка способностей) для игры Deadlock от Valve.
Данная ветка содержит **продвинутую версию генератора на базе Machine Learning (алгоритмы K-Means и FP-Growth)**. В отличие от слепого рандома, эта версия анализирует реальную статистику матчей, выделяет "архетипы" героев и генерирует играбельные, синергирующие билды с контролируемым уровнем хаоса.

## ⚙️ Стек технологий
* **Backend:** Python, Django, Django Channels (WebSockets для мультиплеера).
* **Frontend:** HTML5, CSS3 (Vanilla), Vanilla JavaScript.
* **ML Pipeline:** Python, Pandas, Scikit-learn, MLxtend (FP-Growth), DuckDB (для парсинга Parquet-файлов).
* **База данных:** SQLite (для лобби), JSON/CSV (для ML-моделей).

## 🧠 Как работает ML-Пайплайн (Bake & Ship)
Генерация билдов не нагружает сервер в реальном времени. Мы используем подход "Bake & Ship":
1. **Препроцессинг (Шаг 1):** Скрипты скачивают свежие агрегированные данные по предметам и скиллам из `deadlock-api.com` (или из локальных `.parquet` дампов) и строят векторы сборок для каждого героя.
2. **Кластеризация (Шаг 2):** Алгоритм **K-Means** группирует векторы в "Архетипы" (например: *Фулл Урон с руки*, *Танк через вампиризм*, *Маг*).
3. **Поиск синергий (Шаг 3):** Алгоритм **FP-Growth** находит ассоциативные правила покупок (какие предметы игроки чаще всего покупают вместе в рамках одного архетипа).
4. **Результат:** Пайплайн сохраняет веса и правила в статические файлы `meta_rules.json` и `meta_weights.json`. Сервер просто отдает эти файлы клиенту, а JS-скрипт в браузере мгновенно генерирует билд на их основе.

## 🌡️ Контролируемый Хаос (Температура)
В интерфейсе реализован ползунок "Temperature":
* **Low (Мета):** Алгоритм выдает самый популярный билд строго по выбранному архетипу героя, включая самую частую раскачку способностей.
* **Mid (Адекватный рандом):** Алгоритм допускает нишевые архетипы и подмешивает редкие, но реальные предметы, сохраняя общую синергию.
* **High (Chaos):** Полный хаос. Выбираются абсолютно случайные предметы из всей базы и случайная прокачка способностей.

## ⚠️ Текущие проблемы и зоны для улучшения
* **Интеграция новых героев:** В текущий момент пайплайн опирается на ID героев, которые отдаются по API. Так как API не всегда своевременно обновляется для новых/тестовых персонажей, часть скиллов может генерироваться неточно. В планах — полный переход на анализ сырых логов матчей (`.parquet`).
* **Размер JSON-правил:** По мере увеличения количества синергий `meta_rules.json` может расти в объеме, что потребует сжатия или ленивой загрузки.
* **Отсутствие истории способностей в дампах:** Текущие сырые дампы (`match_player.csv`) содержат отличные данные по предметам, но в них нет истории прокачки скиллов. Поэтому используется гибридный метод (предметы из дампов, скиллы из API).

## 🚀 Запуск проекта
1. Установите зависимости: `pip install -r requirements.txt` (и `pip install duckdb pandas scikit-learn` для ML-пайплайна).
2. Выполните миграции: `python manage.py migrate`
3. Запустите сервер: `python manage.py runserver`
